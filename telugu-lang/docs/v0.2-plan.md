# v0.2 Plan — telugu-lang

This document outlines the planned improvements for the `telugu-lang` v0.2 milestone.

---

## Goals for v0.2

1. **For-loop support** (`కొరకు` — koraku)  
   Add a C-style `for` loop keyword so iterating over ranges is more natural:
   ```tel
   కొరకు (వేరియబుల్ i = 0; i < 10; i = i + 1) {
     చెప్పు(i)
   }
   ```

2. **Arrays and indexing** (`జాబితా`)  
   Introduce array literals and bracket indexing:
   ```tel
   వేరియబుల్ సంఖ్యలు = [1, 2, 3, 4, 5]
   చెప్పు(సంఖ్యలు[0])
   ```

3. **String interpolation** (`${}` inside backtick strings)  
   Allow embedding expressions inside strings using a Telugu-friendly syntax:
   ```tel
   వేరియబుల్ పేరు = "అర్జున్"
   చెప్పు(`నమస్కారం, ${పేరు}!`)
   ```

4. **Null / undefined keyword** (`శూన్యం` — śūnyaṁ)  
   Map to JavaScript `null` for empty/missing values.

5. **Improved error messages**  
   - Show the offending source line with a caret (`^`) pointing to the error column.
   - Provide both Telugu and English error explanations.

6. **Type annotations (optional, structural)**  
   Begin exploring lightweight optional types for documentation purposes:
   ```tel
   పని కూడించు(a: సంఖ్య, b: సంఖ్య): సంఖ్య {
     తిరిగి a + b
   }
   ```

7. **Standard library module** (`stdlib`)  
   Ship built-in Telugu-named wrappers for common operations:
   - `చదవు()` — read input from stdin
   - `వ్రాయి()` — write to stdout (alias for `చెప్పు`)
   - `పొడవు(జాబితా)` — get array/string length
   - `రకం(విలువ)` — get type name

8. **VS Code syntax-highlight extension**  
   Provide a `telugu-lang.vsix` extension with:
   - TextMate grammar for `.tel` files
   - Telugu keyword colorisation
   - Bracket matching for `{}` and `()`

---

## Non-goals for v0.2

- Classes / OOP (deferred to v0.3)
- Module system / imports (deferred to v0.3)
- Async / await (deferred to v0.4)

---

## Acceptance criteria

- All v0.2 language features are covered by unit tests in `src/__tests__/`.
- Each feature has at least one example in `examples/`.
- `docs/grammar.md` is updated to reflect new syntax.
- `docs/keywords.md` lists all new reserved words.
- `mana run` successfully executes all examples without errors.
